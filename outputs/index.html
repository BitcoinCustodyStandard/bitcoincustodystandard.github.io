Resilience Gap Analysis
  • The same device is used for both daily activity and Bitcoin signing operations
  • No personal node; full reliance on third-party servers for transaction verification
  • No verifiable records maintained after custody configuration changes
  • Informal and untested succession and inheritance procedures
  • No structured long-term review or entropy mitigation practices
  • Weak UTXO management, increasing transaction linkage and exposure risk
Resilience Upgrade Path
  • Establish a documented inheritance and succession plan
  • Maintain and update custody records after any configuration change
  • Separate Bitcoin activity from daily usage (dedicated device or isolated user profile)
  • Operate a personal node for independent transaction verification
  • Improve UTXO management to reduce linkage and exposure

Interpretation Reference
Resilience Sufficiency Threshold

Minimum BCRI of 75 required for structural sufficiency. Architectures below this threshold are considered structurally insufficient regardless of other factors.

Resilience Efficiency Frontier

Curve representing peak resilience per unit of complexity. Peaks near CS 4.5 in the benchmark dataset. Beyond the peak, added complexity yields diminishing resilience returns.

Custodial Entropy Effect

All output values are subject to entropy degradation over time. A high initial BCRI does not guarantee sustained resilience without periodic review and succession planning.
